Single Platform Codes

You can embed special text codes in your Single Platform data to unlock special features in your Menu Driver menus.

Categories

Single Platform's menu data is structured with many menus per location, many sections per menu, many items per section, and many price choices per item.

You can organize the menus for a location into categories by adding the category to the name of the menu in square brackets: "Menu Name [Food]". For example, consider these menus:

  • Appetizers
  • Salads
  • Entrees
  • Deserts
  • White Wines
  • Red Wines
  • Champagnes
  • Spirits

You can group those menus into "Food" and "Drinks" by changing the menu names to:

  • Appetizers [Food]
  • Salads [Food]
  • Entrees [Food]
  • Deserts [Food]
  • White Wines [Drinks]
  • Red Wines [Drinks]
  • Champagnes [Drinks]
  • Spirits [Drinks]

Then you will be able to use the categories variable in your menu templates: 

  <ul class="categories">
  <% for category in categories %>
    <li>
      category: <%= category.name %>
      <ul>
      <% for menu in menus %>
        <li>
          menu: <%= menu.name %>
        </li>
      <% end %>
      </ul>
    </li>
  <% end %>
  </ul>

Languages

Indicate the language of a restaurant menu in Single Platform by indicating the ISO 639-1 language code for the language in curly brackets. For example, "Menu Name {en}" for English, or "Menu Name {ar}" for Arabic.

You can combine the codes for categories and languages:

  • Appetizers [Food] {en}
  • Aperitivos [Comida] {es}
  • Entrees [Food] {en}
  • Entradas [Comida] {es}
  • Red Wines [Drink] {en}
  • Vinos Tintos [Bebidas] {es}
  • White Wines [Drink] {en}
  • Vinos Blancos [Bebidas] {es}

Menu Driver will group the categories under a languages varible for your menu templates, so that you can display only the categories appropriate to the current language.
